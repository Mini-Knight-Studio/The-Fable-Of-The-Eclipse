Custom Tools

Loopie Engine

The foundation of the entire project, Loopie is a custom, from scratch game engine designed to bring our game to life. Builded using multiple external libraries to maximize development speed, its architecture was designed to mimic Unity, ensuring a smooth and familiar learning curve for the team.

Key Features

Core Systems: Robust Scene/Entity architecture, an Asset & Resource management system, and an intuitive, user friendly editor.

Graphics & Visuals: Advanced lighting, custom shader support, a dedicated Particle System, and an Animation System featuring skinned mesh support.

Gameplay & Audio: Integrated Audio System, built-in basic collision detection, and a flexible UI system.

Scripting & Workflow: Full C# scripting integration, project management tools, and a semi-automatic game deployment pipeline.

Scene Data Extractor

To overcome our engine's initial lack of a native prefab system, we designed a clever workaround. We used AI to generate this external utility tool based on our specifications. The Scene Data Extractor allowed us to seamlessly move, clone, delete, and preview entities across different scenes, effectively solving a critical workflow gap and saving us massive amounts of development time.

Other tools

Github

Github has been the place where we stored the entire code in two separate repositories (one for the game project and one for the engine), we also made use of its branching system to work simultaneously on a project of our team scale.

Maya

Maya is the main modelling program used for the art development of the game.

Z-Brush

Z-Brush has been the main tool used for detailing the surfaces of our 3D models.

Substance Painter

Substance is the main texturing tool used during the development.

Blender

Blender importing and exporting tools were used to prepare the formatting of our .fbx so that they are ready for our custom engine.

Figma

Figma was used to make sketches and visual schematics for the game.

Hack&Plan

Hack&Plan is the main tool used during development to organize tasks and track the overall progress of the project.

Discord

Discord has been the main tool used for communication, which was done though a custom server prepared for the development of the project.

Google Drive

Google Drive was used as the main storage for all the files created during the development.
